Design a clean, modern developer tool web dashboard called "mroki hub" — a shadow traffic analysis platform for comparing production vs. experimental service responses.

What mroki is:
mroki is a platform for testing service changes by comparing production traffic against shadow deployments in real-time. It mirrors live HTTP traffic to both a production ("live") service and an experimental ("shadow") service, computes JSON response diffs, and lets developers analyze behavior differences — all without impacting real users.

Core concepts:
- A "Gate" is a pair of services: a live (production) URL and a shadow (experimental) URL. Traffic flowing through a gate is forwarded to both services.
- An "Agent" is a transparent HTTP proxy deployed in your infrastructure that intercepts traffic, forwards it to both live and shadow, and captures response differences.
- A "Diff" is the computed difference between the live and shadow JSON responses for each captured request.
- The "Hub" (this UI) is the web interface for managing gates, browsing captured requests, and visualizing diffs side-by-side.

The system flow: Client → mroki-agent (proxy) → forwards to Live Service + Shadow Service → computes diff → sends to mroki-api (REST API + PostgreSQL) → mroki-hub (this web UI) displays it all.
